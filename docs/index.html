Attack Timeline

The attack chain below shows how adversary activity is staged and how the lab validates detection at each step.

01

Reconnaissance

Command: nmap -sS 192.168.117.0/24

Detected by Security Onion through Suricata and Zeek.

02

Brute Force

Linux: Hydra SSH brute force against Ubuntu. Windows: repeated failed logons generating Event ID 4625.

Detected by Wazuh, Ubuntu auth.log, Windows Security logs, and Security Onion SSH traffic patterns.

03

Lateral Movement

Command: net use \\\\192.168.117.130\\c$

Detected through Event 4624, SMB session visibility, and Wazuh correlation.

04

Privilege Escalation

Commands: whoami /priv and runas /user:corp\\administrator cmd

Detected through Event 4672 and Sysmon process telemetry.

05

Persistence

Command: schtasks /create /sc minute /mo 5 /tn "UpdateTask" /tr "cmd.exe"

Detected through Event 4698 and Wazuh persistence-focused rules.

06

Investigation

SOC analyst correlates SIEM alerts, endpoint logs, and Security Onion evidence to validate the attack chain.

Interactive Attack Flow

This section shows how one attack moves through the environment and how different tools observe it.

Reconnaissance

Attack: Kali performs an Nmap scan against the lab network.

Detection: Security Onion observes scan patterns through Suricata and Zeek.

Evidence: Suricata alert + Zeek connection logs.

Brute Force

Attack: Hydra attacks Ubuntu SSH or failed Windows logons are triggered.

Detection: Wazuh monitors Linux auth.log and Windows Event 4625.

Evidence: Wazuh alert + endpoint logs + network evidence.

Lateral Movement

Attack: Administrative SMB access is attempted against DC01.

Detection: Windows Event 4624, Sysmon network activity, and SMB traffic patterns are reviewed.

Evidence: Wazuh alert + Windows logs + Security Onion context.

Persistence

Attack: Scheduled task is created on a Windows system.

Detection: Wazuh detects Event 4698 and Sysmon helps provide execution context.

Evidence: Task creation logs + persistence alert.

SOC Investigation Walkthrough

Example workflow from the lab showing how a brute-force alert against Ubuntu is investigated using multiple evidence sources.

Case Example: SSH Brute Force Against Ubuntu

  • Step 1: Wazuh alert shows repeated SSH authentication failures.
  • Step 2: Analyst checks /var/log/auth.log on Ubuntu to confirm failed password attempts.
  • Step 3: Security Onion is reviewed for repeated SSH connections from the attacker IP.
  • Step 4: Source IP is correlated with Kali attack activity.
  • Step 5: Technique is mapped to T1110 — Brute Force.
  • Step 6: Findings are documented using the incident report template.

Live SOC Investigation Example

This example shows how a SOC analyst investigates a Windows failed logon case from alert to conclusion.

Case

Windows Failed Logon — Event ID 4625

A failed logon alert appears in Wazuh after repeated incorrect authentication attempts against the domain controller.

  • Source host: Win11Client
  • Target: DC01
  • Log source: Windows Security Log
  • Technique: T1110 — Brute Force
Workflow

Analyst Steps

  • Review Wazuh alert details
  • Open Event Viewer and confirm Event ID 4625
  • Identify source IP and username targeted
  • Correlate with Sysmon or related host telemetry
  • Classify the event as credential attack simulation
