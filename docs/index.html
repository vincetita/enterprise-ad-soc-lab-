Detection Stack

Each alert can be investigated from multiple angles: Windows security logs, Sysmon process telemetry, Linux authentication logs, Zeek metadata, and Suricata IDS alerts.

SIEM

Wazuh

Central log collection, rule-based alerting, agent management, and investigation starting point for SOC workflows.

  • Windows Event IDs 4624 / 4625 / 4672 / 4698
  • Linux auth.log ingestion
  • Custom rule support
Endpoint

Sysmon

High-fidelity endpoint visibility on Windows systems including process creation, network connections, and registry modifications.

  • Event ID 1 — Process creation
  • Event ID 3 — Network connections
  • Event ID 13 — Registry changes
Network

Security Onion

Network IDS and metadata platform used to identify scans, suspicious traffic patterns, and connection behavior during attacks.

  • Suricata IDS alerts
  • Zeek connection metadata
  • Packet visibility for investigations

Reconnaissance

02

Brute Force

Linux: Hydra SSH brute force against Ubuntu. Windows: repeated failed logons generating Event ID 4625.

Detected by Wazuh, Linux auth logs, Windows Security logs, and Security Onion SSH traffic analysis.

03

Lateral Movement

Command: net use \\192.168.117.130\c$

Detected through Event ID 4624, SMB session traces, Sysmon network events, and Wazuh correlation.

Privilege Escalation & Persistence

Detected through Event IDs 4672 and 4698 plus Sysmon process telemetry and Wazuh rules.
